Part 1: How Lua Works

Variables

Defining a variable in Lua looks like this:


							name = "Hyacinth"
							health = 100
							is_alive = true

Functions

You can define functions like so:

						
							function add(a,b)
							 return a + b
							end

And call them

						
							add(5,10)

The above function will return 15 when the program is run

Arrays

To define an empty array, we simply write

						
							a = {}

Lua arrays can start at any index, but it's customary to start them at 1 (and most Lua libraries do as well)

Loops

For loops look like

						
							for i = 1, 10 do
							 a[i] = 0
							end

This tells the program to set i to 0, and add 1 to i until it's 10

The ith element of Array a is then filled with a 0

Tables

Tables are super important in Lua, and to making games using the language

For instance, we can write:

						
							player = {}
							player.health = 100
							player.speed = 10
							player.is_dead = false

and then can access those properties using dot notation

						
							if player.health <= 0 then
							 player.is_dead = true
							end

Tables are great for bundling like-functioning data together

So how do we turn any of this into a game?

That's where LoVE comes in

Some important concepts for game scripting

Game Framework

LoVE is a Game Framework, which means that it has a bunch of helper functions and code that allow us to write games far easier than if we didn't have it

Update Loops

Most modern game engines have a built in Update loop

This is a loop that runs every frame and that all of your core logic will go inside of

Draw Loops

Additionally, LoVE has a Draw loop, which does all of the graphics processing for the game

Only code that draws stuff should be put in this function

Coordinate Systems

Delta Time

Computers are REALLY fast these days (our update loop runs about 60 times a second), so we need a way to ensure framerate independence for our games

Delta time is the time (in miliseconds) since the last frame--which allows us to make sure that our game runs at the same speed for everyone
